# AI융합교육원 브로슈어 관리 시스템 사용 설명서

이 문서는 코딩이나 개발 지식이 없는 실무자도 쉽게 브로슈어 내용을 작성하고 웹사이트에 올릴 수 있도록 안내하는 사용 설명서입니다.

---

## 1. 관리자 페이지 접속 및 준비

1. 브라우저 주소창에 관리자 페이지 URL을 입력하여 접속합니다. (예: `http://localhost:3000/2026-ai-guidebook/admin/` 또는 실제 배포된 URL)
2. **대상 레포지토리** 칸에 수정하고자 하는 브로슈어의 레포지토리 이름(예: `template` 또는 `2026-hackathon`)을 정확히 입력하고 [변경 적용]을 누릅니다.
3. 최초 1회, 발급받은 **GitHub 개인 인증 토큰(PAT)**을 입력하고 로그인합니다.

## 2. 문서 속성 (메타데이터) 이해하기

마크다운(.md) 파일의 내용을 적기 전에, 문서 맨 위에 아래와 같은 속성값(Frontmatter)을 적어주어야 시스템이 올바르게 인식합니다.
관리자 페이지에서 **[수정]** 버튼을 누르면 이 속성값들을 화면에서 쉽게 고칠 수 있습니다.

```markdown
---
id: intro
title: 교육원 소개
sidebar_position: 1
theme: blue
description: 부산대학교 AI융합교육원을 소개합니다.
---
```

- **id**: 영문 소문자와 붙임표(-)만 사용하여 짧게 적습니다. 웹사이트 주소가 됩니다. (예: `intro`, `how-to-apply`)
- **title**: 홈페이지와 메뉴에 표시될 실제 제목입니다.
- **sidebar_position**: 왼쪽 메뉴와 메인 화면에 표시될 순서(숫자)입니다.
- **theme**: 카드 배경색입니다. (`blue`, `green`, `purple`, `yellow`, `red`, `slate` 중 택 1)
- **description**: 메인 화면 카드에 나타나는 짧은 설명글입니다.

## 3. 기본 마크다운(Markdown) 작성법

문서를 작성할 때는 마크다운이라는 간단한 기호를 사용합니다.

### 1) 제목 달기
우물정자(`#`)를 띄어쓰기와 함께 사용하면 제목이 됩니다.
```markdown
# 가장 큰 제목 (문서당 1개만 권장)
## 두 번째로 큰 제목
### 세 번째로 큰 제목
```

### 2) 글씨 강조 및 목록
```markdown
**이 글씨는 굵게 표시됩니다**
*이 글씨는 기울임꼴로 표시됩니다*

- 점으로 시작하는
- 글머리 기호 목록입니다.
  - 띄어쓰기 두 번을 하면
  - 안쪽으로 들어갑니다.

1. 숫자로 시작하는
2. 번호 매기기 목록입니다.
```

### 3) 구분선과 링크
```markdown
---
(가로줄 구분선이 생깁니다)

[클릭할 글자](https://www.pusan.ac.kr)
```

## 4. 디자인 꾸미기 (특수 기능)

마크다운 기본 기능 외에도 브로슈어를 예쁘게 꾸밀 수 있는 전용 컴포넌트들을 제공합니다. **이 기능들은 대소문자와 띄어쓰기를 정확히 지켜야 합니다.**

### 1) 알림 상자 (Callout)
안내, 팁, 경고 등을 예쁜 색상 박스로 감싸서 보여줍니다.
```markdown
:::info 공지사항
여기에 공지 내용을 적습니다.
:::

:::tip 꿀팁
알아두면 좋은 정보입니다.
:::

:::warning 주의
주의사항을 적습니다.
:::
```

### 2) 형광펜 (Highlight)
글자 배경색을 칠해 강조합니다.
```markdown
이 문장은 <Highlight color="#ef4444">빨간색</Highlight>으로 강조됩니다.
(기본은 파란색: <Highlight>파란색 강조</Highlight>)
```

### 3) 링크 버튼 (HeroButton)
클릭 가능한 예쁜 버튼을 만듭니다.
```markdown
<HeroButton href="https://pusan.ac.kr">학교 홈페이지</HeroButton>
<HeroButton href="/docs/intro" outline={true}>안내서 홈 (테두리만)</HeroButton>
```

### 4) 다단 나누기 (Columns 레이아웃)
화면을 세로로 2등분 또는 3등분하여 내용을 나란히 배치합니다. `<Col>` 태그 위아래로는 반드시 한 줄씩 띄워야 합니다.

#### 2단 나누기 (반반)
```markdown
<Columns>
  <Col>
  
  **왼쪽 구역입니다.**
  원하는 내용을 적으세요.
  
  </Col>
  <Col>
  
  **오른쪽 구역입니다.**
  이미지나 글을 넣으세요.
  
  </Col>
</Columns>
```

#### 3단 나누기
```markdown
<Columns cols={3}>
  <Col> 첫 번째 칸 </Col>
  <Col> 두 번째 칸 </Col>
  <Col> 세 번째 칸 </Col>
</Columns>
```

### 5) 탭 메뉴 (Tabs)
메뉴를 클릭할 때마다 내용이 바뀌게 만듭니다. 이 기능을 쓸 때는 맨 위에 `import` 문장을 꼭 적어주세요.

```markdown
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="tab1" label="탭 1" default>첫 번째 내용</TabItem>
  <TabItem value="tab2" label="탭 2">두 번째 내용</TabItem>
</Tabs>
```

## 5. 관리자 페이지 기능 가이드

- **문서 업로드**: 작성 완료한 `.md` 파일을 'MD 파일 업로드' 영역에 드래그하여 올리면, 즉시 시스템에 반영되고 1~2분 뒤 웹사이트에 배포됩니다.
- **순서 및 제목 수정**: 기존 파일의 순서(1번, 2번...)가 꼬였거나 제목을 바꾸고 싶을 때는 파일 목록 옆의 **[수정]** 버튼을 누릅니다. 모달창에서 번호나 제목을 고치고 **[저장 및 배포]**를 누르면 끝입니다. 파일을 삭제하고 다시 올릴 필요가 없습니다.
- **파일 삭제**: 불필요한 문서는 **[삭제]** 버튼을 누르면 즉시 지워집니다. 모든 문서를 다 지우면 웹사이트가 고장(배포 에러)날 수 있으니 반드시 최소 1개의 문서(예: `id: intro` 속성을 가진 문서)는 남겨두어야 합니다.
